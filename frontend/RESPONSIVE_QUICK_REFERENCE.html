📱 Responsive Design Quick Reference

Essential snippets and best practices for responsive web design

1. Viewport Meta Tag (CRITICAL!)

This MUST be in every responsive website's <head> section:

<meta name="viewport" content="width=device-width, initial-scale=1.0">
This tells browsers to render at the device's actual width and set initial zoom to 100%

2. Media Query Breakpoints

Device Min Width Max Width CSS Query
Mobile 320px 480px @media (max-width: 480px)
Tablet 481px 768px @media (max-width: 768px)
Small Laptop 769px 1024px @media (max-width: 1024px)
Desktop 1025px ∞ @media (min-width: 1025px)

3. Responsive CSS Grid

Auto-fit Grid (Recommended)

/* Automatically adjusts columns based on content width */
.grid {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(300px, 1fr));
    gap: 1rem;
}
/* Mobile: 1 column
   Tablet: 2 columns
   Desktop: 3-4 columns (automatic!) */

Media Query Grid

/* Manual control with media queries */
.grid {
    display: grid;
    grid-template-columns: 1fr;
    gap: 1rem;
}

@media (min-width: 768px) {
    .grid { grid-template-columns: repeat(2, 1fr); }
}

@media (min-width: 1024px) {
    .grid { grid-template-columns: repeat(3, 1fr); }
}

4. Responsive Typography

Using REM Units

html {
    font-size: 16px; /* Base size */
}

body {
    font-size: 1rem; /* 16px */
}

h1 {
    font-size: 2.5rem; /* 40px */
}

p {
    font-size: 1rem; /* 16px */
    line-height: 1.6;
}

Scaling Font Sizes

/* Mobile first */
h1 { font-size: 1.5rem; }
p { font-size: 0.9rem; }

/* Tablet and above */
@media (min-width: 768px) {
    h1 { font-size: 2rem; }
    p { font-size: 1rem; }
}

/* Desktop and above */
@media (min-width: 1024px) {
    h1 { font-size: 2.5rem; }
    p { font-size: 1.1rem; }
}
font-size: 16px; /* ❌ Avoid pixels for font */
font-size: 1rem; /* ✅ Use relative units */

5. Responsive Images

Basic Responsive Image

<img src="image.jpg" 
     alt="Description"
     style="max-width: 100%; height: auto;">

Background Images

.hero {
    background-image: url('image.jpg');
    background-size: cover;
    background-position: center;
    background-attachment: fixed;
}

Picture Element (Multiple Sources)

<picture>
    <source media="(min-width: 1024px)" srcset="large.jpg">
    <source media="(min-width: 768px)" srcset="medium.jpg">
    <img src="small.jpg" alt="Description">
</picture>

Lazy Loading

<img src="image.jpg" alt="Description" loading="lazy">
Always use max-width: 100%; height: auto; to prevent distortion

6. Responsive Container

/* ✅ CORRECT - Flexible container */
.container {
    width: 100%;
    max-width: 1200px;
    margin: 0 auto;
    padding: 0 1rem;
}
/* ❌ WRONG - Fixed width */
.container {
    width: 1200px;
    margin: 0 auto;
}
Use max-width instead of fixed width for flexibility

7. Responsive Flexbox

.flex-container {
    display: flex;
    flex-wrap: wrap;
    gap: 1rem;
}

.flex-item {
    flex: 1 1 300px; /* min-width: 300px, grows/shrinks */
}

/* Mobile: Stack vertically (flex-wrap: wrap) */
/* Desktop: Side by side automatically */

8. Touch-Friendly UI Elements

Buttons (Minimum 44x44px)

button {
    padding: 0.75rem 1.5rem; /* Comfortable padding */
    min-height: 2.75rem;     /* At least 44px */
    font-size: 1rem;
    border-radius: 0.5rem;
    cursor: pointer;
    transition: all 0.3s ease;
}

button:hover {
    background-color: #007bff;
    transform: translateY(-2px); /* Subtle lift */
}
Touch targets must be at least 44x44px for accessibility

9. Common Mistakes to Avoid

Fixed Widths

.container { width: 960px; } /* ❌ Breaks on smaller screens */
.container { max-width: 960px; width: 100%; } /* ✅ Flexible */

Pixel-Based Font Sizes

body { font-size: 16px; } /* ❌ Doesn't scale */
body { font-size: 1rem; } /* ✅ Scales with user preferences */

Horizontal Overflow

img { width: 1200px; } /* ❌ Causes scrolling */
img { max-width: 100%; height: auto; } /* ✅ Fits device */

Not Testing Touch

Always test on actual mobile devices, not just desktop browser emulation

10. CSS Custom Properties for Responsiveness

:root {
    --primary-color: #667eea;
    --spacing-xs: 0.5rem;
    --spacing-md: 1rem;
    --spacing-lg: 2rem;
    --font-size-base: 1rem;
}

body {
    color: var(--primary-color);
    font-size: var(--font-size-base);
}

.card {
    padding: var(--spacing-lg);
    margin-bottom: var(--spacing-md);
}

/* Update for mobile */
@media (max-width: 768px) {
    :root {
        --spacing-lg: 1rem; /* Reduced spacing on mobile */
        --font-size-base: 0.9rem;
    }
}

11. Testing Responsiveness

Browser DevTools

  1. Press F12 to open Developer Tools
  2. Click the device toggle icon (📱) in the top-left
  3. Select different devices to test
  4. Resize the viewport to test breakpoints

Real Device Testing

Always test on actual devices:
  • iPhone, iPad, Android phones/tablets
  • Various screen sizes
  • Different network speeds
  • Landscape and portrait orientations

12. Performance Optimization

/* ✅ Good Practices */

/* Use relative units */
margin: 1rem;
padding: 1.5rem;

/* Combine selectors */
h1, h2, h3 { font-family: Arial; }

/* Use min-width for mobile-first */
@media (min-width: 768px) { /* Fewer rules */ }

/* Lazy load images */
<img loading="lazy">

/* Minimize repaints */
will-change: transform;
transform: translateZ(0);
